RAID : redundant array of inexpensive disk / redundant array of independent disk.
-- multiple disk are connected to make a single logical unit
-- provides redundancy and improved performance or both compared to a single disk
-- different raid levels give us different levels of redundancy/performance
-- can be managed by software from OS or by RAID controller in hard ware

RAID 0 : AKA stripped set
-- better performance, no redundancy
-- data is split evenly accross all the set accross disk, same data split and not copied hence no redundancy
-- if any disk in the set fails, then all the data is lost.

-- the read and write is faster because there is no copying going on, and just write and read, the write and read is done simultaneously in multiple disks
-- the more number of disks the more better performance, write and read is done parallelely in these disk hence increasing performance.

-- you can use different disk type, RPM speed and size in same storage system but not in same raid group.
-- using different capacity disks, the usable size is limited to smallest disk size.

RAID 1 : AKA Mirrored set
-- the same data is written to both disks in the set, giving high redundancy, but less performance
-- If one disk fails, the data is stored in other disk, high redundancy
-- write performance is not improved, because writing in 2 disks
-- read performance is improved, as reads can be serviced by both disks.

RAID 4 :

Data Striping: Data is split across multiple disks (like RAID 0).
Dedicated Parity Disk: One disk is used exclusively to store parity information (used for recovery in case of disk failure).
Minimum Disks Required: At least 3 disks — 2 for data, 1 for parity
can tolerate upto 1 disk failure.

RAID 5 :

Same as RAID 4 but with distributed parity.
Data can be recreated from parity if any one of the disk fails.
read performance increased as multiple disk concurently service read, and write performance also increased.
can tolerate upto 1 disk failure.
There no single parity disk, instead there are parity blocks.

RAID 6 :

can tolerate upto 2 disk failure.
2 parity blocks distributed accross disks.
data can be recreated through parity if 2 disks fail.
R/W performance increased

Hybrid RAID :

multiple raid levels nested for better performance.
RAID 10 -- multiple raid 1 mirrored sets are nested inside a RAID 0 striped set
RAID 0+1 -- multiple RAID 0 sets are nested into RAID 1 mirror
RAID 50( 5 + 0 ) -- multiple RAID 5 parity sets are nested into RAID 0 stripped set

SAN vs NAS

NAS ( network attached storage ) : provides file level access to storage resources.
-- the management of the file system resides with the remote storage system.
-- NAS typically uses the already existing data network infra

SAN ( storage area networks ) : provides block level access to storage resources.
-- the client manages the its allocated space on storage system as if it was a local hard disk
-- the file system management resides with the client.
-- gives highest control over storage devices, very low level access.
-- SAN has traditionally got a seperate dedicated storage network infra.

Boot from SAN : 
You can boot your OS from a SAN, having no hard drive at the local server, all the bootable files are stored in the SAN.
no storage is reuired at all, without local storage you can bootup, this creates diskless servers.
There can be boot from SAN, but not NAS, it not possible.

======================================================================= Virtualization =========================================================================

Virtualization : creating VM on a already present hardware.

host : a physical server which has RAM, memory, storage interface, network interface, plain device, nothing on top of it.
hypervisor : this is installed on the plain host, its like a agent, which manages how much storage, RAM, memory, network bandwidth is to be given to each VM on the shared infra
exsi -- is by VMware
VM : virtual machine's is being created on the hypervisor, multiple VM's can be created upon hypervisor and each VM has its own Operating system.

Types of hypervisor:

1. Baremetal hypervisor, TYPE 1 hypervisor : its created/installed on the plain hardware server, and it takes ownership of the hardware, there is no undrlying OS below this.
eg : VMware Exsi, citrix, microsoft Hyper-v.

2. TYPE 2 hypervisor : hypervisor is installed/running on your local system, its installed on an already existing OS, it works on that OS, and it creates multiple VM's on that.
eg : VMware workstation, parallel for MAC, VMware playstation.

Hypervisor is the platform on which the VM's run.

Food groups:
1. CPU -- the hypervisor allocates vCPU to each VM, each VM requires specific vCPU cores, which had to be bought, after buying its allocated from physical CPU and is converted to vCPU and allocated further.

2. Memory -- the host's physcial memory is being shared among the guest VM's, sometimes the multiple guest VM's can share same memory, not at the same time, but when free.
          -- you can also reserve the memory for your own paticular VM.
		  -- the hypervisor smartly allocates host memory accross guest VM's

3. Network -- every VM has its own vNIC's presented to it as physcial NIC
		   -- virtual switch is present on each hypervisor, in case if the internal VM's want to connect and a virtual switch is connected pNIC to connect to actual switch
		      to connect to internet.

4. Storage -- vSCSI is given to each VM, 
           -- scsi commands from VM are passed to hypervisor and then passed through storage adapters to storage area.
		   -- decoupling is used for mobility as there is no direct contact between storage and VM's its only through hypervisor, which allows to for VM migration.
		   
Benifits of virtulization:
1. consolidation
2. Flexibility

Live state : refers to the VM's running condition, capturing its memory, CPU activity, network traffic, and storage usage as it executes on the host machine.

vMotion -- live migration of the VM's from one to other host without having any downtime, can be used for many reasons, like hypervisor upgrade and reboot, all the storage,
network and vNIC are also same on the other host no changes at all.

cluster -- groups multiple hosts together makes a cluster.
Load balancing -- VM's from one host can be shifted / migrated temporarily or permanently to balance load on host, VM's can be shifted to any host inside the cluster.
memory bit map -- during the migration of server from one host to other the changing memory is written in bit map and is transferred accross the other host, this makes
                migration faster.
