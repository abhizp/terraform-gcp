tar command -----------
tar -cvzpf file_name ./directory_to_be tarred.
c -- create
v -- verbose output
z -- zip
f -- specify file_name
p -- preserve permissions

tar -xvzf file_name -- to extract to the same directory structure.

tar --same-owner -xvzf file_name -- to unzip with same owner permissions as it was wn tarred
tar -xvzf file_name --no-same-owner -- to make yourself owner of the files when untarred

dd command -------------

dd -- copy convert -- to copy the entire disks data all at once

dd if=/dev/sda of=/mnt/storage/sda.img status=progress -- to copy to the other disk and also see the progress
if = input file
of = output file
/mnt/storage/sda.img -- created this file in the different disk and stores the image there, it literally copies eerything it even copies empty space too, the whole disk size

dd if=/dev/sda of=/mnt/storage/sda.img status=progress bs=1M -- to increase speed
bs -- to tell dd command how much block size to copy at once, default is 512 bytes

dd if=/dev/sda of=/mnt/storage/sda.img status=progress bs=1M conv=sync,noerror -->
conv -- to specify options, sync -- confirms if every bit is written line by line like perfect clone, noerror -- continues even if it encounters error.

dd if=/dev/sda bs=1M | gzip -c > /mnt/storage/sda11.img -- to zip the file and create the image to save some space

to restore the file:
gunzip -c /mnt/storage/sda11.imgz | dd of=/dev/sdd1 bs=1M status=progress -- to restore the disk to original state.

if not zipped

dd if=/mnt/storage/sda11.img of=/dev/sdd1 bs=1M status=progress -- before restoring always unmount the disk.

rsync command -----------

rsync [OPTIONS] SOURCE DESTINATION - syntax
you can copy files inside the server and also outside the servers.
it only transfers differences between source and destination
used to syncronize files

inside server:

rsync -azurhP ./Downloads ./backups -- inside server

a -- archieve mode -- it preserves all the attributes like owner, timestamp etc..
z -- zip it and transfer, and also unzips on destination
u -- update, its skips files on destination if they are newer than source
r -- recursively sync, usefull if there are sub directories
P -- shows progress.
h -- human readable format

outside server :

rsync -azurvhe ssh ./Downloads root@server_name:/root/directory/
e ssh <command> -- specifies the shell to use ssh to do remote file transfer. -- it tells to open a ssh tunnel and transfer it over that tunnel,
v -- verbose mode

exclude/include few files while transfer:

rsync -azurvhe ssh ./Downloads root@server_name:/root/directory/ --exclude "*.pdf" --include ".*" -- this is to exclude all pdf files and trasnfer the rest.
rsync -azurvhe ssh ./Downloads root@server_name:/root/directory/ --include "*.pdf" --exclude ".*" -- this is to only transfer pdf files and exclude everything else.

Dry run before actually executing:

rsync -azurvhe ssh ./Downloads root@server_name:/root/directory/ --include "*.pdf" --exclude ".*" --dry-run
A -- to preserve and transfer ACL permissions too.
g -- to preserve and transfer group permissions
o -- to preserve and transfer owner permissions

Journalctl command:
1. journalctl -- to get all the log info, about everything in the system, services, hardware, etc..
2. journalctl --since "YYYY-MM-DD HH:MM:SS" --until "YYYY-MM-DD HH:MM:SS" -- to see logs between a specific time range.
3. journalctl -r -- to reverse the order and show the new entries first.
journalctl -n 2 -- recent 2 entries
journalctl | grep Centaur -- filter by keyword
journalctl -p warning -- display logs based on priority -- (e.g., emerg, alert, crit, err, warning, notice, info, or debug)
journalctl -o verbose -- you can get options too, like verbose mode
journalctl --list-boots -- to view information about system boots
journalctl -g "keyword1" -g "keyword2" -- filtering by keywords
2. journalctl -f -- to display the live/following log entries
tail -f -- also works in similar way.
